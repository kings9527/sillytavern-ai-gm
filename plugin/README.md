## 快速开始

### 1. 安装插件

```bash
cd SillyTavern/public/scripts/extensions/third-party
git clone https://github.com/kings9527/sillytavern-ai-gm.git
ln -s sillytavern-ai-gm/plugin ../../../plugins/ai-gm
```

重启 SillyTavern，确保 `config.yaml` 中 `enableServerPlugins: true`。

### 2. 启用 AI-GM

1. 打开 SillyTavern 右侧扩展栏
2. 找到 **AI-GM 克苏鲁跑团** 并展开
3. 勾选 **启用 AI-GM 功能**
4. 设置战役 ID（默认 `default`）
5. 勾选 **连接后自动启动**（可选）

### 3. 开始游戏

插件会自动加载内置模组 **「阿卡姆之夜」**，你扮演一名调查员进入密斯卡托尼克大学图书馆开始探索。

## 基础操作

### 输入指令

在聊天框输入自然语言，AI-GM 会自动解析你的意图：

| 你想做的事 | 示例输入 |
|-----------|---------|
| 移动 | `去地下室` `进入图书馆` `走大门` |
| 调查 | `检查书架` `观察地面` `看看桌子` |
| 交谈 | `和图书管理员说话` `问博士关于仪式的事` |
| 使用物品 | `用钥匙开门` `读那本日记` |
| 技能检定 | `检定图书馆使用` `侦查` `聆听` |
| 战斗 | `攻击邪教徒` `开枪` `使用格斗` |
| 逃跑 | `逃跑` `撤退` `离开这里` |

### 场景动作按钮

每个场景下方会自动生成可用的动作按钮：
- **移动**: 可前往的出口
- **交谈**: 场景中可对话的 NPC
- **检查**: 可交互的物品
- **战斗**: 进入战斗（如果场景有敌人）

## 战斗系统

### 战斗流程

1. **遭遇敌人**: 进入有敌人的场景时自动触发战斗
2. **先攻判定**: 根据 DEX 值和 d100  roll 决定行动顺序
3. **玩家回合**: 选择攻击、技能、使用物品或逃跑
4. **敌人回合**: 敌人自动行动（规则驱动 + LLM 增强决策）

### 战斗指令

| 指令 | 示例 |
|------|------|
| 攻击 | `攻击邪教徒` `用格斗攻击` |
| 使用技能 | `闪避` `使用射击` |
| 逃跑 | `逃跑` `撤退` |

### 伤害计算

- 普通攻击: 武器伤害 + 伤害加值 (DB)
- 大成功 (≤5): 最大伤害
- 大失败 (≥96): 自伤 1 点

### 伤害加值 (DB) 表

| STR+SIZ | DB |
|---------|-----|
| ≤ 64 | -2 |
| 65-84 | -1 |
| 85-124 | 0 |
| 125-164 | +1d4 |
| 165-204 | +1d6 |
| ≥ 205 | +2d6 |

## NPC 交互

### NPC 态度系统

每个 NPC 有独立的态度状态，影响其行为：

| 态度 | 行为倾向 |
|------|---------|
| 友好 (friendly) | 乐于帮助、透露信息 |
| 中立 (neutral) | 保持礼貌、有限合作 |
| 敌对 (hostile) | 攻击或威胁 |
| 恐惧 (afraid) | 逃跑、求饶、回避 |
| 敌对警觉 (hostile_alerted) | 积极攻击 |
| 逃跑中 (hostile_fleeing) | 试图逃离 |

### 影响 NPC 态度的因素

- **信任 (Trust)**: 高信任 → 更友好，低信任 → 更敌对
- **恐惧 (Fear)**: 高恐惧 → 逃跑或崩溃
- **怀疑 (Suspicion)**: 高怀疑 → 回避敏感话题

### 秘密与线索

NPC 可能知道秘密。通过提升信任、使用特定话题或成功检定，可以让他们透露线索。

## 存档与读档

AI-GM 支持 **5 个存档位**（SQLite 持久化 + 内存备份）。

### 自动保存

- 每次场景切换自动保存到内存
- 手动存档会写入 SQLite 数据库

### 手动存档

在设置面板中：
1. 输入存档位（1-5）
2. 输入存档标签（如「图书馆入口」）
3. 点击 **保存**

### 读档

选择存档位后点击 **读取**，会恢复到保存时的场景和状态。

## 模组加载

### 模组格式

AI-GM 支持两种模组格式：

1. **JSON 格式**: 完整的结构化模组文件
2. **Markdown 格式**: 带 YAML 前言的 Markdown 文件，便于写作和版本控制

### Markdown 模组结构示例

```markdown
---
id: arkham-night
name: 阿卡姆之夜
version: 1.0.0
system: coc7e
author: 你的名字
---

# 场景: 图书馆大厅

**id**: library_hall
**start**: true

密斯卡托尼克大学的图书馆大厅灯火通明。古老的书架高耸入云，空气中弥漫着纸张和霉味混合的气息。

**氛围**: 安静的、略带不安的

## 出口
- [前往地下室](basement) — 需要: 找到钥匙
- [离开图书馆](exit)

## NPC
- [图书管理员](npcs/librarian)

## 物品
- [奇怪的日记](items/diary)

## 战斗
- [邪教徒](npcs/cultist)

### 事件: 书架倒塌
**触发**: action: inspect
**效果**:
- narration: 书架轰然倒塌，灰尘弥漫！
- sanity_loss: 1d3
```

### 加载模组

将 `.md` 或 `.json` 模组文件放入 `plugin/modules/` 目录，在设置面板中选择模组 ID 并重新启动。

## 常见问题

### Q: 插件无法启用？
- 检查 `config.yaml` 中 `enableServerPlugins: true`
- 检查浏览器控制台是否有错误
- 确认 `node_modules/better-sqlite3` 已安装（如果需要用 SQLite）

### Q: NPC 不说话？
- 检查 NPC 是否存活（HP > 0）
- 检查 NPC 态度是否为 `dead`
- 检查 LLM API 是否可用（如果使用 LLM 增强对话）

### Q: 战斗不触发？
- 确认场景配置中有 `combat: enabled: true`
- 确认场景有 `enemies` 列表
- 检查是否已在一个活跃战斗中

### Q: 存档丢失？
- 如果 `better-sqlite3` 未安装，存档仅保存在内存中，重启后丢失
- 运行 `npm install better-sqlite3` 启用持久化

### Q: 骰子结果不随机？
- 检查是否使用了 `DiceRoller` 缓存（解析缓存不会缓存 roll 结果，只缓存结构）
- 每次 roll 都是新的随机值
