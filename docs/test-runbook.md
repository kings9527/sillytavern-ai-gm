# 周六测试流程（06-13）

## 周六测试流程（06-13）

## 交付方式
**完整安装包**：`sillytavern-ai-gm.zip`（约 5-10 MB）

包含：
- `install.sh` / `install.ps1` — 一键安装脚本
- `ai-gm-plugin.zip` — AI-GM 插件文件
- `README-INSTALL.md` — 安装指南

### 安装步骤（Windows 11）
1. **安装 Node.js**：下载 [Node.js 22 LTS](https://nodejs.org/)，安装时勾选 "Add to PATH"
2. **下载** `sillytavern-ai-gm.zip`，解压到桌面或任意文件夹
3. **右键 PowerShell** → 以管理员身份运行
4. 进入解压目录：`cd C:\Users\你的用户名\Desktop\sillytavern-ai-gm`
5. 执行安装：`.\install.ps1`
6. 脚本会自动下载 SillyTavern、安装依赖、放入 AI-GM 插件
7. 启动：`cd SillyTavern && npm start`
8. 浏览器打开 `http://localhost:8000`
9. 扩展面板 → AI-GM → 「新建战役」

### 如果已有 SillyTavern
只下载 `ai-gm-plugin.zip`，解压到 `SillyTavern\` 目录（覆盖 `plugin\` 和 `public\`），重启即可。

## 玩家测试流程（30-60 分钟）

### 步骤 1：创建战役（2 分钟）
1. 在 SillyTavern 中打开 **AI-GM 面板**（右侧扩展按钮）
2. 点击 **「新建战役」**
3. 选择模组：「密斯卡托尼克档案」
4. 确认角色卡已加载（调查员属性：HP 12、SAN 60、技能值）

**预期结果**：
- 显示战役创建成功
- 场景 1「密斯卡托尼克大学图书馆」加载
- 看到场景描述、2 个 NPC、可用动作按钮

---

### 步骤 2：场景 1 — 图书馆（8-10 分钟）
**目标**：获取第一个线索（古籍缺页）

1. **与老管理员对话** → 点击 NPC 头像 → 选择「询问古籍」
   - 预期：管理员回复关于古籍失踪的信息
2. **图书馆使用检定** → 点击「搜索书架」→ 1d100 投骰
   - 技能值 25，≤25 成功
   - 预期：成功 → 发现古籍缺页；失败 → 描述没有收获
3. **前往教授办公室** → 点击出口按钮

**验证点**：
- [ ] NPC 对话有内容，不是空字符串
- [ ] 检定自动触发，显示结果（成功/失败）
- [ ] 成功获得线索 1：`clue_missing_page`

---

### 步骤 3：场景 2 — 教授办公室（8-10 分钟）
**目标**：获取第二个线索（教授日记）

1. **与研究生助手对话** → 了解教授失踪时间
2. **侦查检定** → 搜索办公桌 → 1d100 投骰
   - 技能值 40
   - 预期：成功 → 发现日记残页；失败 → 助手提醒你去公墓看看
3. **心理学检定** → 观察助手表情 → 判断是否在隐瞒
   - 预期：成功 → 助手紧张，提到 "第七页"
4. **前往阿卡姆公墓**

**验证点**：
- [ ] 对话有分支（根据检定结果不同）
- [ ] 获得线索 2：`clue_diary_fragment`
- [ ] 关键信息 "第七页" 出现

---

### 步骤 4：场景 3 — 阿卡姆公墓（10-12 分钟）
**目标**：获取第三个线索（邪教证据）

1. **聆听检定** → 夜间墓地 → 听到地下声音
   - 技能值 30，≤30 成功
   - 预期：成功 → 发现地下入口；失败 → 自行发现（保底设计）
2. **场景切换**：发现地下入口 → 自动进入场景 4

**验证点**：
- [ ] 夜间场景氛围不同（描述有变化）
- [ ] 检定失败有保底机制（不卡关）

---

### 步骤 5：场景 4 — 地下密室（战斗/潜行分支）（10-15 分钟）

**分支 A：战斗路线**
1. 发现邪教徒在进行仪式
2. 选择「战斗」→ 进入战斗系统
3. 回合制：
   - 玩家回合：攻击 → 检定格斗/射击 → 结算伤害
   - 敌人回合：邪教徒攻击 → 玩家 HP 减少
4. 击败所有敌人 → 进入场景 5

**分支 B：潜行路线**
1. 选择「潜行」→ 潜行检定（DEX 基础）
2. 成功 → 绕过敌人 → 获取邪教证据 → 进入场景 5
3. 失败 → 被迫进入战斗

**验证点**：
- [ ] 战斗系统正常启动（先攻计算、回合切换）
- [ ] 伤害结算正确（HP 减少、死亡判定）
- [ ] 敌人 AI 有行为（攻击/逃跑逻辑）
- [ ] 战斗结束清理正确（combat_state 清空）
- [ ] 潜行分支有效

---

### 步骤 6：场景 5 — 终局（5-8 分钟）
**目标**：根据理智值触发结局

**触发条件**：进入场景 5 时
- 如果已有 3 个线索：正常结局
- 如果缺少线索：SAN 损失 + 1d3

**理智检定**：
1. 看到邪教真相 → 强制 SAN 检定（POW 基础）
2. 结果：
   - 成功（≤POW）：SAN 损失 1
   - 失败（>POW）：SAN 损失 1d6

**结局分支**：

| 条件 | 结局 | 描述 |
|------|------|------|
| SAN ≥ 30 | **理智逃脱** | 你逃离了密室，但那些知识将永远纠缠你 |
| SAN 10-29 | **精神崩溃** | 真相压垮了你，你的意识在恐惧中消散 |
| SAN < 10 | **被邪教捕获** | 你加入了他们，成为下一个仪式的祭司 |

**验证点**：
- [ ] 结局根据 SAN 值正确触发
- [ ] 结局描述完整显示
- [ ] 战役自动结束（无法继续行动）

---

### 步骤 7：存档/读档测试（额外 5 分钟）
1. 在场景 3 时手动保存 → 选择存档位 1
2. 继续玩到场景 5 结局
3. 读档 → 选择存档位 1
4. 预期：回到场景 3，状态完全恢复

**验证点**：
- [ ] 存档成功，有反馈提示
- [ ] 读档后场景正确、HP/SAN 正确、线索正确
- [ ] 存档列表显示场景/回合/时间信息

## 如果测试失败

| 失败场景 | 处理 |
|----------|------|
| 插件加载失败 | 检查 `plugin/index.js` 语法，查看浏览器控制台 |
| 场景卡住 | 检查 `state-machine.js` 场景切换逻辑 |
| 检定不触发 | 检查 `rule-engine.js` 调用和前端事件绑定 |
| 战斗崩溃 | 检查 `combat-tracker.js` 空值处理 |
| 存档丢失 | 检查 `campaign.js` save/load 逻辑 |
