## 7 天里程碑

### Day 1 · 06-07（周六）— NPC 决策引擎
**目标**: 将骨架补全为可运行的 NPC 决策系统

- 完成 `npc-decision.js` 规则驱动逻辑：
  - 基于态度（neutral/friendly/hostile）的行为选择
  - 战斗中的 AI 决策（攻击/逃跑/使用技能）
  - 场景中的 NPC 交互（对话/线索提供/战斗触发）
- 集成到状态机的 `interact` 和 `combat` 分支
- 测试：「阿卡姆之夜」中 NPC 能否正确对话和战斗

**预计工时**: 21:00 + 周六白天 2h（可选）

### Day 2 · 06-08（周日）— 模组解析器 + 边界修复
**目标**: Markdown 模组可解析，场景事件边界安全

- 完成 `module-parser.js` Markdown 解析：
  - 场景标题、描述、出口、NPC 列表、事件定义
  - 从 Markdown 生成 JSON 模组数据
- 修复场景事件多次触发的边界处理
- 修复战斗结束时清理 `combat_state` 的边界检查

**预计工时**: 21:00 + 周日白天 2h（可选）

### Day 3 · 06-09（周一）— 存档系统 + 测试修复
**目标**: 存档可靠，测试框架可运行

- 修复存档读取后恢复 `combat_state` 完整性
- 存档验证：读取时检查必要字段，缺失则报错
- 修复 `test/index.js` 内存溢出问题（Node 循环引用排查）
- 编写核心模块单元测试（状态机、规则引擎、战斗）

**预计工时**: 21:00

### Day 4 · 06-10（周二）— 类型安全 + 输入验证
**目标**: 代码健壮，防 XSS

- JSDoc 类型注解到所有公共 API
- 参数验证中间件（统一 schema 验证）
- 输入 sanitization（防止 XSS 注入到 narration）
- ESLint 配置（基础规则）

**预计工时**: 21:00

### Day 5 · 06-11（周三）— 集成测试
**目标**: 「阿卡姆之夜」端到端跑通

- 完整跑团测试：创建战役 → 场景 1 → 检定 → 场景 2 → 战斗 → 结局
- 记录所有报错和异常行为
- 修复发现的 bug（预期 3-5 个）
- 前端 UI 细节调整（根据测试反馈）

**预计工时**: 21:00

### Day 6 · 06-12（周四）— 性能优化 + 最终完善
**目标**: 流畅运行，无明显卡顿

- 状态机性能优化（减少不必要的对象拷贝）
- 前端渲染优化（减少 DOM 操作）
- 错误边界：添加 try-catch 到关键用户路径
- 日志系统完善：关键事件持久化到文件
- 最终 bug 修复

**预计工时**: 21:00

### Day 7 · 06-13（周五）— 测试准备日
**目标**: 为周六跑团测试做准备

- 上午：最终检查，确认所有模块正常运行
- 下午：准备测试环境（SillyTavern + AI-GM 插件 + 测试模组）
- 晚上：记录测试 checklist，准备反馈表格
- 周六：正式单人跑团测试

**预计工时**: 21:00 + 周五白天 1h（准备环境）
